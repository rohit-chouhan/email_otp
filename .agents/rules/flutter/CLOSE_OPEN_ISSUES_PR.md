trigger: always_on
description: Close Open Issues and Pull Requests
---

For every open Issue and Pull Request:

- Investigate and determine whether it is still relevant to the current codebase.
- If the issue has already been fixed in a previous release, provide a closure comment explaining when and how it was resolved.
- If the issue is a duplicate, provide a closure comment referencing the original issue.
- If the issue is outdated, no longer reproducible, or no longer relevant, provide a professional closure comment explaining the reason.
- If the issue requires code changes, implement the fix, add/update tests, update documentation if needed, and provide a closure comment summarizing the changes.
- If a Pull Request is valid, review it, resolve conflicts if possible, merge or recommend merging, and provide a review summary comment.
- If a Pull Request is outdated, superseded, abandoned, or no longer compatible with the current codebase, provide a detailed closure comment explaining why it is being closed.
- Every Issue and Pull Request must have a ready-to-post GitHub comment generated by the agent.

Goal:
- No open Issue or Pull Request should be left unreviewed.
- Close all Open Issues with Proper Comments
- Every open Issue and Pull Request must end with one of the following outcomes:
  - Fixed and Closed
  - Duplicate and Closed
  - Outdated and Closed
  - Invalid and Closed
  - Merged
  - Superseded and Closed

OPEN_ISSUES.md file:
- Find all Open Issue and PR details in `OPEN_ISSUES.md` file in current project

Final Objective:
The repository should have zero unresolved actionable Issues and Pull Requests. The agent must either fix, merge, or provide a justified closure recommendation for every open Issue and Pull Request, along with professional GitHub comments for each action taken.
